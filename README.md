## O Desafio: Interface de Assinaturas "Zero"

A **Zero** é um ecossistema de streaming voltado para a convergência entre **games e entretenimento**. O objetivo deste projeto é desenvolver uma interface de planos que não seja apenas funcional,
mas que atue como uma solução estratégica para os conflitos de interesse de três áreas fundamentais da empresa:

### Estratégia Mobile-First & Performance
Atendendo ao **Diretor de Tecnologia**, o projeto é desenvolvido com foco total em performance para redes 3G/4G. 
* **O Problema:** 75% dos usuários utilizam smartphones intermediários com conexões instáveis.
* **A Solução:** Implementação de técnicas de otimização como *lazy loading*, uso de imagens em formatos modernos (WebP) e um JavaScript enxuto para garantir que o tempo de carregamento não afaste o cliente.

### O Equilíbrio de Stakeholders (Negócio vs. UX)

A interface precisa ser o ponto de convergência entre visões distintas:

| Setor | Objetivo Principal | Requisito para o Desenvolvimento |
| :--- | :--- | :--- |
| **Financeiro** | Aumentar o fôlego de caixa e reduzir o *churn*. | Destacar o **Plano Anual** como a opção mais vantajosa, sem ocultar o mensal. |
| **Marketing** | Reforçar o valor "Premium" da marca e gerar *referrals*. | Uso de elementos visuais de alto impacto e aproveitamento do potencial de indicação do **Plano Mensal**. |
| **Tecnologia** | Garantir acessibilidade e velocidade. | Evitar scripts complexos e assets pesados que prejudiquem o carregamento em redes móveis. |

### Nossa Abordagem Técnica
Para satisfazer a todos, nossa equipe adotou as seguintes premissas:
1. **Design de Alto Impacto (Marketing):** Uso de CSS moderno para efeitos visuais leves que transmitam a sensação "Premium" sem depender de imagens gigantescas.
2. **Arquitetura de Conversão (Financeiro):** Design psicológico focado em destacar o Plano Anual através de contraste e *social proof*.
3. **Código Otimizado (Tech):** JavaScript focado em manipulação eficiente do DOM e lógica de alternância (Toggle) rápida entre planos.
